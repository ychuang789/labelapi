## 一、需求

2.0 版的改良版本，修改貼標任務 API, worker 降低貼標使用資源與加速貼標時間。

## 二、目標

+ 貼標任務 worker 使用批次處理方法，降低記憶體使用率。
+ 貼標任務追蹤表新增欄位儲存結果輸出統計資訊。
+ 貼標任務追蹤表新增錯誤訊息與檢查點資訊，降低專案程式維護成本。
+ 統一各 API response 格式
